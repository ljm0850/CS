- 네트워크 : 다수의 시스템을 전송 매체로 연결해 구성한 시스템들의 집합체



## 계층모델

- 여러 호스트들을 서로 연결해서 통신하려면 연결 방식의 표준화가 필요

### OSI 7계층 모델

1. 물리계층(Physical Layer)  : 비트전송
2. 데이터 링크 계층(Data Link Layer) : 오류 확인
3. 네트워크 계층(Network Layer) : 데이터 경로 설정

---

4. 전송계층(Transport Layer) : 데이터 전송

---

5. 세션 계층(Session Layer)

6. 표현 계층(Presentation Layer)

7. 응용 계층(Application Layer)



#### 물리계층(Physical Layer, 1계층)

- **비트전송** : 데이터를 0과 1의 전기적 신호로 바꿔 전송 || 신호를 비트로 복원

- **전송 매체 규정** : 데이터가 흐르는 물리적 매체(케이블, 무선 주파수 등)를 정의
  - 예: 전기 신호(UTP 케이블), 빛 신호(광섬유), 전파(Wi-Fi 등)
- **전기적/기계적/절차적 특성 정의** : 하드웨어 간의 물리적 연결 규칙을 정함



#### 데이터 링크 계층(Data Link Layer, 2계층)

- **물리적으로 연결된 두 장치 간에 ‘오류 없는 데이터 전송’을 보장하는 계층**



- **오류 감지 및 수정 (Error Detection / Correction)**

  - 물리 계층에서 전송 중 신호 손실이나 잡음이 생길 수 있음 → 이를 검출하고 복구.
  - 예: 패리티 비트, CRC(Cyclic Redundancy Check) 등 사용.

- **프레임화 (Framing)**

  - 상위 계층(3계층)에서 받은 데이터를 **‘프레임(Frame)’ 단위**로 나눔.

  - 프레임에는 **주소(MAC 주소)** 와 **오류검출코드(FCS)** 등이 포함됨.

  - ```
    프레임 : 데이터링크 계층에서 다루는 기본 단위(상위 계층에서 내려온 패킷을 프레임으로 감쌈)
    
    [ 헤더 | 데이터 | 트레일러(FCS) ]
         ↑주소 정보, 제어 정보 등 포함
    ```

- **흐름 제어 (Flow Control)**

  - 송신 측이 너무 빨리 데이터를 보내 수신 측이 감당 못할 때 조절.

- **매체 접근 제어 (Media Access Control)**

  - 여러 장치가 동일한 통신 매체를 사용할 때, **누가 먼저 전송할지** 결정.
  - 예: CSMA/CD (Ethernet), CSMA/CA (Wi-Fi)
  - Eternet : **유선 LAN 환경**에서 컴퓨터 간 데이터를 프레임 단위로 주고받는 **표준 규격**이며, MAC 주소를 사용해 장치를 구분



#### 네트워크 계층(Network Layer, 3계층)

- **데이터를 목적지까지 전달하기 위해 “어떤 경로로 보낼지”를 결정하는 계층**



- **라우팅 (Routing)**

  - 데이터를 목적지까지 보내기 위해 **최적의 경로(Path)** 를 결정.
  - 라우터(router)가 이 기능을 담당

- **논리적 주소 지정 (Logical Addressing)**

  - 각 장치를 구별하기 위해 **IP 주소**를 부여.
  - (2계층의 MAC 주소는 “물리적 주소”, 3계층의 IP는 “논리적 주소”)
  - 서로 다른 네트워크 간 통신이 가능해짐.

- **패킷 전달 (Packet Forwarding)**

  - 데이터를 **패킷(Packet)** 단위로 전송.

    - ```
      패킷
      [ IP 헤더 | 데이터 ]
      
      IP 헤더에는 출발지 IP 주소와 목적지 IP 주소가 들어있음
      ```

  - 출발지 → 목적지까지 여러 네트워크를 거치며 전달.

- **단편화(Fragmentation)와 재조립(Reassembly)**

  - 네트워크마다 전송할 수 있는 최대 크기(MTU)가 다르므로,

    너무 큰 패킷은 쪼개서 전송하고 수신 측에서 다시 합침.



---



#### 전송계층(Transport Layer, 4계층)

- **송신지와 수신지 간의 ‘종단 간(end-to-end)’ 데이터 전송을 담당하는 계층**



- **신뢰성 있는 전송 보장 (Reliable Transmission)**

  - 데이터가 손상되거나 누락되지 않게 관리

  - 필요 시 재전송, 순서 정렬, 흐름 제어 등을 수행

- **포트 번호(Port Number) 관리**

  - 같은 장치(같은 IP) 안에서도 여러 프로그램이 동시에 통신할 수 있도록, 각 애플리케이션을 구분하는 “번호표” 역할
  - 예: HTTP → 포트 80, HTTPS → 443, FTP → 21, SSH → 22

- **세그먼트화(Segmentation) 및 재조립(Reassembly)**

  - 상위 계층(5~7)에서 받은 데이터를 작은 단위로 쪼개 전송
  - 수신 측에서는 다시 원래 데이터로 조립

- **흐름 제어 (Flow Control)**

  - 수신자가 감당할 수 있는 속도에 맞춰 송신 속도를 조절

- **혼잡 제어 (Congestion Control)**

  - 네트워크가 혼잡할 때 데이터 전송 속도를 줄여 과부하 방지

  

##### TCP / UDP 프로토콜

| **TCP (Transmission Control Protocol)** | 연결 지향형, 신뢰성 보장, 순서 제어, 재전송 지원          |
| --------------------------------------- | --------------------------------------------------------- |
| **UDP (User Datagram Protocol)**        | 비연결형, 빠르지만 신뢰성 보장 없음, 실시간 서비스에 적합 |



| **구분**  | **TCP**                                      | **UDP**                        |
| --------- | -------------------------------------------- | ------------------------------ |
| 연결 방식 | 연결 지향(Connection-oriented)               | 비연결(Connectionless)         |
| 신뢰성    | 높음 (재전송, 순서제어)                      | 낮음 (보내고 끝)               |
| 전송 속도 | 느림                                         | 빠름                           |
| 오버헤드  | 큼 (헤더 20바이트 이상)                      | 작음 (헤더 8바이트)            |
| 전송 단위 | 세그먼트                                     | 데이터그램                     |
| 사용 예시 | 웹(HTTP/HTTPS), 이메일(SMTP), 파일 전송(FTP) | 스트리밍, 게임, 음성/영상 통화 |
| 대표 포트 | 80(HTTP), 443(HTTPS)                         | 53(DNS), 123(NTP)              |

---



#### 세션 계층(Session Layer, 5계층)

- 하드웨어보다는 소프트웨어 연결 관리쪽에 가까움
- 통신을 “**대화(세션)**”처럼 관리하는 계층, **연결의 생성, 유지, 종료를 담당**하고, 오류로 끊긴 경우 **재연결(복구)** 도 담당
- **‘언제 대화 시작하고, 언제 끝내는지’를 관리**



| **기능**                              | **설명**                                                     |
| ------------------------------------- | ------------------------------------------------------------ |
| **세션 설정 (Session Establishment)** | 통신을 시작하기 위한 “대화방”을 여는 과정→ 예: 로그인 시 서버와 세션 생성 |
| **세션 유지 (Session Maintenance)**   | 연결이 끊기지 않도록 상태를 유지하고 관리                    |
| **세션 동기화 (Synchronization)**     | 전송 중 오류가 발생했을 때 **중간 저장점(Checkpoint)** 에서 다시 시작 가능 |
| **세션 종료 (Termination)**           | 대화가 끝나면 자원 해제 및 연결 종료                         |



#### 표현 계층(Presentation Layer, 6계층)

- 데이터를 “**사람과 시스템이 이해할 수 있는 형태로 변환**”
- **형식, 인코딩, 암호화 등을 통일해주는 계층**



| **기능**                                               | **설명**                                                     |
| ------------------------------------------------------ | ------------------------------------------------------------ |
| **① 데이터 변환 (Translation)**                        | 서로 다른 시스템 간 문자 인코딩이나 데이터 구조 차이를 해결  |
| **② 데이터 압축 (Compression)**                        | 전송 효율을 높이기 위해 데이터 용량을 줄임 예: 동영상 스트리밍, 이미지 전송 |
| **③ 데이터 암호화 / 복호화 (Encryption / Decryption)** | 보안을 위해 데이터 암호화 후 전송, 수신 시 복호화 수행 예: HTTPS 통신 시 SSL/TLS 암호화 |



#### 응용 계층(Application Layer, 7계층)

- 사용자가 직접 접하는 **애플리케이션과 네트워크가 상호작용하는 계층**
- **웹 브라우저, 이메일 클라이언트, FTP 클라이언트 등이 네트워크를 사용할 수 있게 해주는 계층**



- **사용자와 네트워크 간 인터페이스 제공**

  - 애플리케이션이 네트워크 자원(서버, 데이터)과 소통할 수 있게 함

  - 예: 브라우저가 HTTP 요청 보내기, 이메일 클라이언트가 SMTP 요청 보내기

- **네트워크 서비스 제공**

  - 파일 전송, 이메일 송수신, 원격 접속 등

  - 응용 계층이 제공하는 서비스가 바로 우리가 “인터넷을 사용한다”고 느끼는 것들

- **프로토콜 규정**

  - 애플리케이션 간 통신을 위한 규칙과 포맷 정의

  - 예: HTTP, HTTPS, FTP, SMTP, DNS 등
